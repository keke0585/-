# 视频研判系统 (Smart Video Analysis)

## 1. 项目概述
这是一个基于目标检测（YOLO）和人脸捕捉（MediaPipe/OpenCV）的视频研判辅助系统，旨在对视频画面进行智能分析、活动目标（人体/车辆）裁切以及人脸提取。系统全面重构为 B/S 架构，专为离线环境和 Windows 7+ 兼容交付设计，推荐在 Windows 10/11 + Python 3.8.10 环境中打包部署。

## 2. 核心功能及实现
- **多模式检测**: 支持“仅检测人”、“仅检测车辆”、“人车双检”动态实时切换 (依靠前沿 YOLO 模型及目标追踪)。
- **批量处理队列**: 支持批量导入视频文件，并以队列顺序自动分析。拥有进度状态追踪、智能跳帧极速研判模式及强制跳过功能。
- **智能去重**: 引入基于空间距离及时间跨度的人脸/人体碰撞去重防抖机制，大幅减少无效截图。
- **硬件加速**: 支持智能跨平台算力调度，自动检测并切换 CUDA(Nvidia) / MPS(Apple Silicon) / CPU 推理，自带环境检测降级策略。
- **双模联动人脸捕捉**: 提供 MediaPipe 与 OpenCV Haar 级联的双模动态人脸捕捉（优先高精度，无环境时回退至兜底方案）。

## 3. 系统架构
- **解耦式后端 (Python)**: `eyes_modern.py` 负责算法核心逻辑与并发任务调度。内置轻量级无依赖 `HTTPServer` 提供数据接口。采用 `threading` 后台分析与主服务数据完全解耦。
- **前端页面 (Web UI)**: 位于 `web/` 目录。
    - **图标系统**: 使用 SVG Symbol 矢量图库，确保离线与无网环境完美呈现。
    - **交互逻辑**: 原生 Vanilla JavaScript + CSS 变量实现的高响应式、沉浸式暗色工业风界面。
- **系统级集成**: 核心系统交互（如原生弹窗与文件资源选取）均通过 Python Subprocess 挂载独立 `tkinter` 进程桥接执行，彻底杜绝 UI 阻塞。
- **流媒体传输**: 前后端状态通过轻量级 REST API 以及 `STATE` 全局内存字典同步，图像渲染流基于高性能原生 MJPEG (Multipart X-Mixed-Replace) 流式输出。

## 4. 品牌视觉与资源
- **应用图标**: 极简工业美学设计（实心镜头/青色核心），基于 `draw_icon.py` 矢量代码绘制并一键生成高规多分辨率 `app.ico` 原生图标集。
- **动态寻址**: 所有外部私有资源（HTML 网页、Icon、模型权重 `.pt` 库等）均经由 `get_resource_path()` 获取，自动透穿 PyInstaller `_MEIPASS` 虚拟释放目录。

## 5. 打包与部署环境
- **工具链组件**: 包含 `build.py`, `patch.py`, `win7_compat_patch.py`, 及 `check_env.py`。支持通过脚本一键修复依赖池并生成全封装独立 `.exe` 执行文件。
- **动态链接与系统兼容**: 交付包专向 Windows 7 及隔离虚拟机底层环境深层优化（如手动注入加载 `torch` 等 DLL 环境路径），强烈要求依赖锁定：必须于 Windows 10/11 + Python 3.8.10 内执行标准打包流程，严禁升级 PyTorch 基线。

## 6. 开发规范
遵循中文原生架构师规范，高内聚低耦合。坚持防御性编程原则，秉承“能用原生和成熟方案绝不造次轮子”理念，不引入如 PyQt 及复杂型 Web 框架即可实现商业级视觉部署要求。
